Portfolio

Synapse: SQ3R Reading Tool

Description:

Developed an interactive web-based application to enhance active reading and comprehension through the SQ3R methodology (Survey, Question, Read, Reflect, Review). The tool allows users to upload and view PDFs, manage study tasks, create notes, generate flashcards, and download session data for effective learning management.

Key Features:

  • Interactive Interface: Seamlessly integrates a checklist, Q&A section, and note-taking features.
  • Flashcard Generator: Converts study notes into flashcards for efficient review.
  • Local Storage Integration: Preserves user data across sessions without requiring a backend.
  • PDF Interaction: Provides an intuitive way to upload, display, and interact with PDF files.
  • Customizable Flashcards: Enables users to review questions and answers with an easy-to-navigate interface.

Technologies Used:

  • Frontend: HTML, CSS, JavaScript
  • Backend: PHP for file uploads
  • Additional Tools: PDF.js for rendering PDFs

Impact:

  • Enhanced user engagement with structured learning strategies.
  • Encouraged active comprehension through the SQ3R method, proven to improve retention and study efficiency.

Future Development Goals:

  • Introduce multi-user capabilities with user authentication.
  • Expand support for additional file formats.
  • Implement server-side storage for long-term data management.

Serial Position Effect Experiment

Description:

Developed an interactive web application to study the serial position effect—a cognitive phenomenon where individuals recall the first and last items in a sequence more effectively than those in the middle. The tool presents users with randomized word lists and records their recall accuracy, providing insights into memory and learning patterns.

Key Features:

  • Randomized Word Lists: Generates unique sequences for each session to ensure unbiased results.
  • Timed Recall Tasks: Implements countdown timers to simulate real-world memory challenges.
  • Data Visualization: Displays user performance metrics, highlighting primacy and recency effects.
  • User-Friendly Interface: Designed with intuitive navigation to facilitate seamless user experience.

Technologies Used:

  • Frontend: HTML, CSS, JavaScript
  • Backend: Python with Flask framework
  • Data Visualization: Chart.js library

Impact:

  • Enhanced understanding of cognitive biases related to memory recall.
  • Provided a practical tool for psychology students and educators to explore the serial position effect.

Future Development Goals:

  • Incorporate additional cognitive tests to broaden the scope of the application.
  • Enable user authentication for personalized tracking of performance over time.
  • Implement advanced statistical analysis features to offer deeper insights into user data.

DVapor: Register Discrepancy Checker

Description:

Designed and developed a web application for a small business to identify and resolve discrepancies between financial register files and Chase bank records. This project utilized the Software Development Life Cycle (SDLC) methodology, incorporating client feedback to deliver a practical solution for reconciling transaction data efficiently.

Key Features:

  • File Upload System: Enabled secure and user-friendly upload of register and bank files for analysis.
  • Automated Data Cleaning: Processed and cleaned data files, handling non-date entries and formatting issues.
  • Discrepancy Detection: Identified unmatched transactions and generated detailed reports for review.
  • Match Finder: Mapped transactions from both files to find matches based on dates and amounts.
  • Downloadable Reports: Provided users with ready-to-download CSV files for matched transactions, discrepancies, and processed data.

Technologies Used:

  • Backend: Python with Flask framework
  • Data Processing: Pandas for data cleaning and analysis
  • Frontend: HTML, CSS, JavaScript
  • File Handling: Local storage with CSV processing

Impact:

  • Reduced time spent on manual data reconciliation by automating key processes.
  • Improved financial accuracy and accountability for the business.
  • Provided a scalable and reusable tool for ongoing transaction analysis.

Features and Skills Showcased:

  • Client Collaboration: Worked closely with the client to identify pain points and tailor the solution to meet specific business needs.
  • Data Processing Expertise: Utilized Python's Pandas library for advanced data cleaning, formatting, and analysis.
  • Full-Stack Development: Integrated backend data processing with a user-friendly frontend interface.
  • SDLC Methodology: Applied structured development practices to ensure a robust and maintainable solution.
  • Problem-Solving: Addressed challenges such as handling incomplete data and ensuring compatibility with different file formats.

Future Development Goals:

  • Incorporate cloud storage for secure file handling and remote access.
  • Enable multi-user support with role-based access control.
  • Add real-time reconciliation features for enhanced efficiency.
